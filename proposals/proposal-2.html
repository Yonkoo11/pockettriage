§ II · Architecture

One narrow path. No cloud fallback. No telemetry to anyone.

CHW symptom (+ optional photo) │ ▼ infer.py builds WHO-IMCI grounded chat │ ▼ ollama @ 127.0.0.1:11434 → gemma4:e2b (5.1 B · Q4_K_M) │ ▼ raw string _extract_first_json · _validate_shape ← tolerant coerce │ ▼ { tier · pathway · reasoning · confidence } safety.apply_safety_layer R13 · danger-sign → force PINK R14 · conf < 0.4 → append "Escalate to medical officer" R15 · adult case → refusal │ ▼ Gradio renders tier · pathway · reasoning · safety flags
§ III · Safety

Four invariants the model cannot override.

PocketTriage treats the classifier as fallible. The safety layer runs after the model. 17 unit tests cover it (laptop/test_safety.py).

  1. R13 · PINKIf the symptom text or model reasoning contains any WHO IMCI general danger sign, tier is forced to PINK. Negation-aware: "no chest indrawing" does not fire.
  2. R14 · FLOORIf confidence < 0.4, append "Escalate to medical officer" — low-confidence cases never bypass a human.
  3. R15 · ADULTAdult-specific conditions (chest pain in adult, stroke, pregnancy emergency) return a refusal. Paediatric scope, hard boundary.
  4. R16 · BANNERA non-dismissable banner — "Decision support based on WHO IMCI 2014. Does NOT replace clinical judgment. Paediatric only (2 months – 5 years)." — is part of the layout, not a popup.

§ V · Run locally

Clean clone to first triage — about fifteen minutes.

# 1. Install Ollama
brew install ollama
ollama serve &

# 2. Pull the model (one-time, ~7 GB)
ollama pull gemma4:e2b

# 3. Run PocketTriage
git clone https://github.com/yonkoo11/pockettriage
cd pockettriage/laptop
python3.12 -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt

# 4. Verify Phase 1 Gate
python eval_runner.py     # 4 / 4 PASS

# 5. Launch the UI
python app.py             # → http://127.0.0.1:7860
